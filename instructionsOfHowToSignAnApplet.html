How to sign an applet (with Maven)

  • update your pom.xml with the following maven plugins: maven-jar-plugin, maven-assembly-plugin, maven-jarsigner-plugin
  • here is my pom.xml for your reference
  • the maven-jar-plugin is used to build the jar at the package phase (easiest plugin to understand)
  • the maven-assembly-plugin is used to build one huge jar with dependencies (i.e. log4j )at the package phase
  • the maven-jarsigner-plugin is used to sign the jar at the package phase

    take a special note of the configuration section in the pom.xml for this plugin and notice the keystore element.
    Basically, it means you have to have a directory called keystore with a file called signing-jar.keystore.
    Here is my directory structure Feel free to use mine. Its self-signed, meaning it is insecure.

  • to run maven, you can do it from the command line (assuming you have maven installed): "mvn package" or ...
  • from Eclipse: Right click Project->Run As->Maven build...(and type package in the textbox Goals)
  • here is an Eclipse launch file that does the same thing as above

    hint, you can run Eclipse Launch Files by clicking on that file and then F11

Other tidbits of advice: test your applet locally first: i.e. file:////index.html
in your index.html file, make sure that your applet tag code uses your ...jar-with-dependencies.jar and has the correct applet code
See my applet tag example here
If you are getting a self-signed error when you run your applet, modify your security settings in Java Console (Exeption Site List). Java Console can be launched by: Windows Start->Configure Java
